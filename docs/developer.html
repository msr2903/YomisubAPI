Smart Lookup Algorithm

To resolve homonyms (e.g., æœ¬ as "book" vs "origin", words like "can" vs "out of stock"), we use a weighted scoring system:

Common Entry +10 Prioritizes common words (common=True)
Reading Match +20 Prioritizes exact reading match (e.g. ã»ã‚“ vs ã‚‚ã¨)
Usually Kana +15 Prioritizes "usually kana" entries for hiragana input (fixes ã“ã¨)

        if score > best_score:

Metadata Extraction

The API extracts rich metadata from dictionary entries to provide context in the tags field:
